00 — Strategy
The four-phase arc.

Donna's launch follows a narrative arc borrowed from the best tech launches — not a product dump, but a story. We build intrigue before we explain. We explain before we sell. We sell by making people feel they'd be foolish to miss out. Every post has one job: pull the next reader one step closer to early access.

Phase 01
Intrigue
Days 1–7
We don't explain the product yet. We establish the pain. Who is Donna? What's broken about your morning? We name the character before we reveal the solution.
Phase 02
Education
Days 8–16
Feature-by-feature reveals. We show the briefing, the commitment tracker, the heartbeat monitor. We make people feel what it would be like to have this. Real copy. Real mockups.
Phase 03
Desire
Days 17–23
Use cases. Social proof. The "why now" argument. We close the gap between "interesting" and "I need this." We make the ideal user see themselves in every post.
Phase 04
Launch
Days 24–30
Early access is live. We drive sign-ups, create urgency, answer objections publicly, and celebrate the community. Every post has one CTA: get early access.

Brand voice on social

✓  Donna says
"Every morning, one briefing. Everything that matters. Nothing that doesn't."
Confident, precise, understated — like someone who's already read the room
Short sentences. Active voice. No hedging.
Reference the Suits character sparingly — it's an easter egg, not a crutch
Name real pains: the missed follow-up, the forgotten promise, the cold relationship
Treat the reader as someone already intelligent — just overwhelmed
✗  Donna never says
"Game-changing AI solution" — vague, overused, meaningless
Robotic AI-speak: "leverage," "synergy," "streamline your workflow"
Feature lists without context or emotion
"Are you struggling with inbox overload?" — leading questions are weak
Overselling: "The only tool you'll ever need" — it sounds desperate
Emoji overload or exclamation marks — Donna is composed, not excitable

An intelligence layer that takes actions you didn't ask for is a liability, not a tool. DAY 16 X / TWITTER
The Spec Thread
THEME: For the technical audience — how donna actually works
1/5For the technically curious: here's how donna actually works under the hood.
2/5Ingestion runs overnight via Trigger.dev background jobs.

Every integration (Gmail, Calendar, Slack, Notion) is read via OAuth — tokens vaulted in Nango, never touched by our application layer.

Content is sanitised before it reaches any AI prompt. No raw user data ever hits a model directly.
3/5Two-pass AI pipeline:

Pass 1 → Claude Haiku: cheap, fast extraction — pull commitments, flag relationships, detect sentiment signals

Pass 2 → Claude Sonnet: accurate scoring and briefing generation — the reasoning layer that decides what actually matters
4/5The briefing is generated, ranked by a 5-dimension priority engine, and delivered via Telegram Bot API.

Every factual claim in the briefing carries a source_ref — a pointer back to the originating message. We never show an AI claim without a citation.

Built on Next.js 16, Supabase (pgvector for semantic search), Vercel.
5/5We're publishing an architecture deep-dive on the blog next week.
