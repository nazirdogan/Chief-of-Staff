16 The Technical Deep-Dive
Twitter/X
MON · 9AM
TWITTER/X THREAD
1/5 — For the technically curious: here's how donna actually works under the hood. —— 2/5 — Ingestion runs overnight via Trigger.dev background jobs. Every integration (Gmail, Calendar, Slack, Notion) is read via OAuth — tokens vaulted in Nango, never touched by our application layer. Content is sanitised before it reaches any AI prompt. No raw user data ever hits a model directly. —— 3/5 — Two-pass AI pipeline: Pass 1 → Claude Haiku: cheap, fast extraction — pull commitments, flag relationships, detect sentiment signals Pass 2 → Claude Sonnet: accurate scoring and briefing generation — the reasoning layer that decides what actually matters —— 4/5 — The briefing is generated, ranked by a 5-dimension priority engine, and delivered via Telegram Bot API. Every factual claim in the briefing carries a source_ref — a pointer back to the originating message. We never show an AI claim without a citation. Built on Next.js 16, Supabase (pgvector for semantic search), Vercel. —— 5/5 — We're publishing a full architecture deep-dive on the blog next week. For now: donna.app — early access now live.
